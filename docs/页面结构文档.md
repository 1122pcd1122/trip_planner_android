# Trip Planner Android 应用 - 完整页面结构文档

## 一、应用架构概览

**技术栈**：
- 架构：MVVM（视图层 / ViewModel层 / 模型层）
- UI框架：Jetpack Compose (Material3)
- 本地数据库：Room
- 网络请求：Retrofit + OkHttp
- 异步处理：Kotlin Coroutines + Flow

**主题系统**：
- 支持深色/浅色模式切换
- 使用 `AppColors` 数据类统一管理颜色
- 通过 `CompositionLocal` 提供主题颜色
- `ThemeManager` 管理主题偏好设置

## 二、页面列表

### 1. 主页面 - TravelPlannerScreen
**文件**：`TravelPlannerScreen.kt`
**功能**：旅行规划主屏幕，包含5个Agent Tab

**结构**：
```
TravelPlannerScreen
├── TabRow (5个Tab)
│   ├── 🎯 全部 (ALL)
│   ├── 🌤️ 天气 (WEATHER)
│   ├── 🏨 酒店 (HOTEL)
│   ├── 🏛️ 景点 (ATTRACTION)
│   └── 🍽️ 餐厅 (RESTAURANT)
│
└── 内容区域 (根据Tab切换)
    ├── WeatherTabContent
    │   ├── WeatherInputCard (城市选择器 + 日期选择器 + 开始规划按钮)
    │   └── 天气列表 / 加载状态 / 空状态
    │
    ├── HotelTabContent
    │   ├── HotelInputCard (城市选择器 + 日期选择器 + 开始规划按钮)
    │   ├── AgentFilterToolbar (搜索 + 评分过滤 + 排序)
    │   └── 酒店列表 / 加载状态 / 空状态
    │
    ├── AttractionTabContent
    │   ├── AttractionInputCard (城市选择器 + 日期选择器 + 开始规划按钮)
    │   ├── AgentFilterToolbar (搜索 + 评分过滤 + 排序)
    │   └── 景点列表 / 加载状态 / 空状态
    │
    ├── RestaurantTabContent
    │   ├── RestaurantInputCard (城市选择器 + 日期选择器 + 偏好标签 + 开始规划按钮)
    │   ├── AgentFilterToolbar (搜索 + 评分过滤 + 排序)
    │   └── 餐厅列表 / 加载状态 / 空状态
    │
    └── AllInOneTabContent
        ├── AllInOneInputCard (城市选择器 + 日期选择器 + 偏好标签 + 开始规划按钮)
        ├── SummaryContent (行程汇总展示)
        │   ├── 出行建议卡片
        │   ├── 酒店推荐列表
        │   └── 每日行程列表
        └── ItineraryEditor (编辑模式)
            ├── 每日行程编辑
            ├── 酒店编辑
            └── 添加/删除天数
```

**关键组件**：
- `CitySelector`：城市选择器（支持搜索 + 列表选择）
- `DateRangePicker`：日期范围选择器
- `AgentFilterToolbar`：Agent筛选工具栏
- `ItineraryEditor`：行程编辑器
- `WeatherCard`：天气卡片
- `PoiCard`：POI卡片（酒店/景点/餐厅通用）

## 四、数据模型

### 1. AgentType
```kotlin
enum class AgentType {
    ALL, WEATHER, HOTEL, ATTRACTION, RESTAURANT
}
```

### 2. PoiType
```kotlin
enum class PoiType {
    HOTEL, RESTAURANT, ATTRACTION
}
```

### 3. PoiModel
```kotlin
data class PoiModel(
    val name: String,
    val rating: String,
    val price: String,
    val distance: String,
    val latLng: LatLng,
    val desc: String,
    val priceRange: String = "",
    val featureDish: String = "",
    val poiType: PoiType = PoiType.ATTRACTION
)
```

### 4. DetailType
```kotlin
sealed class DetailType {
    abstract val data: Any
    data class HotelDetail(override val data: Any) : DetailType()
    data class AttractionDetail(override val data: Any) : DetailType()
    data class RestaurantDetail(override val data: Any) : DetailType()
}
```

## 五、状态管理

### 1. MainViewModel
- 管理用户输入（目的地、天数、偏好）
- 管理UI状态（Idle/Loading/Success/Error）
- 协调各Agent数据获取
- 支持日期范围输入

### 2. FavoriteViewModel
- 管理收藏数据
- 支持添加/删除收藏
- 按类型筛选

### 3. HistoryViewModel
- 管理历史行程数据
- 支持搜索/删除

### 4. DetailViewModel
- 管理详情数据加载
- 支持缓存机制

## 六、网络与缓存

### 1. TripRepository
- 网络请求层
- 支持所有Agent接口

### 2. CachedTripRepository
- 带缓存的数据仓库
- 24小时缓存有效期
- 优先读取缓存，失效后请求网络

### 3. NetworkMonitor
- 网络状态监听
- 断网提示
